GBasm

A Game Boy CPU Emulator written in Java

Overview

GBasm is a Game Boy CPU emulator written in Java, designed to emulate the behavior of the Z80-like CPU used in the original Game Boy. The emulator simulates the CPU’s functionality by interpreting and executing opcodes one step at a time. It provides a modular design to facilitate easy expansion and testing.

Features

  • CPU Initialization and State: Initializes the Game Boy CPU registers and resets them to match the Game Boy's startup values.
  • Memory Handling: Handles memory reading and writing with bounds checking. Supports reading from and writing to memory locations.
  • Opcode Handling: Supports a few opcodes such as LD A, d8, NOP, HALT, and arithmetic instructions like ADD A, B, SUB A, B.
  • Fetch-Execute Cycle: Implements the fetch-execute cycle to simulate the CPU's behavior.
  • Memory Boundaries Checking: Ensures that reads and writes to memory addresses are within valid bounds (0x0000 to 0xFFFF).
  • Basic Debugging: Prints debugging messages for instruction execution, allowing you to monitor the CPU’s state.

Current Limitations

While the emulator can run a small set of instructions, many opcodes are still unimplemented. Additionally, flag management and interrupt handling need to be implemented for full functionality. Future work will also include more advanced memory and I/O management.

Planned Features

  • Full opcode set support
  • Flag management for various instructions
  • Interrupt handling
  • Input/output handling for user interaction
  • Enhanced debugging tools

Getting Started

  1. Clone the repository: git clone https://github.com/KaidaQ/GBasm.git
  2. Compile the project: javac -d bin src/*.java
  3. Run the emulator: java main.Emulator

Once the emulator is running, you can modify the Memory class or add new opcodes to simulate additional functionality.
